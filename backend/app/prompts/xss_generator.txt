Тип XSS и сложность задания:
- beginner:      "reflected" — параметр напрямую вставляется в HTML, нет фильтров, простой <script>alert()</script>
- intermediate:  "reflected" или "stored" — есть базовый фильтр (замена <script>, case check), нужен обход через event handler или SVG
- advanced:      "dom" или "stored" — DOM-based XSS через location.hash или innerHTML, или stored XSS с обходом CSP/фильтра

РЕКОМЕНДАЦИИ CVE → XSS механика (применяй, если в CVE есть указанный CWE):
- CWE-79 (XSS) → reflected или stored XSS без фильтра — стандартная уязвимость
- CWE-80 (Basic XSS) → тег <script> вставляется напрямую в страницу
- CWE-87 (Alternate XSS syntax) → фильтр блокирует <script>, нужен обход через <img onerror=...>, <svg onload=...>
- CWE-116 (Improper encoding) → данные не экранируются при вставке в HTML атрибут — обход через закрытие кавычки
- CWE-83 (XSS in URI) → уязвимый параметр в href/src атрибуте — javascript: URI схема
- CWE-692 (CSP bypass) → stored XSS с обходом Content-Security-Policy

Правила для payload_solution:
- Для reflected XSS: payload должен быть URL-encoded значением параметра (e.g. `<script>document.cookie</script>` или `"><img src=x onerror=alert(document.cookie)>`)
- Для stored XSS: payload вставляется в поле ввода которое сохраняется и отображается другим пользователям
- Для DOM XSS: payload передаётся через location.hash или URL fragment и вставляется через innerHTML
- Флаг ВСЕГДА хранится в document.cookie под именем "flag" и должен быть получен через XSS
