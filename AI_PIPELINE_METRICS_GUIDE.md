# 🤖 HackNet AI Pipeline — Справочник метрик

## Метрики и их назначение

### 📊 **Метрики Эффективности Генерации**

| Метрика | Назначение | Где используется | Целевое значение |
|---------|-----------|------------------|------------------|
| **Avg Generation Time (ms)** | Измеряет скорость генерации одного варианта задачи | Оптимизация задержек API | < 3000ms |
| **Avg Tokens/Task** | Средний расход токенов на генерацию | Контроль стоимости LLM вызовов | Зависит от типа |
| **Total Variants Generated** | Общее количество сгенерированных вариантов | Мониторинг объемов | > 0 |
| **Completion Rate (%)** | Процент батчей, успешно завершивших генерацию | Надежность пайплайна | > 95% |

---

### ✅ **Метрики Качества**

| Метрика | Назначение | Где используется | Целевое значение |
|---------|-----------|------------------|------------------|
| **Pass Rate (Binary) %** | % вариантов, прошедших все бинарные проверки вознаграждения | Оценка пригодности к продакшену | > 70% |
| **Avg Quality Score /10** | Среднее качество сгенерированных задач (оценка LLM) | Обратная связь в GRPO | > 7.5 |
| **Selection Rate** | Кол-во вариантов, отобранных для публикации | Контроль выхода в продакшн | 1 из N лучших |
| **Total Batches** | Всего запросов на генерацию | Статистика использования | Растет > 0 |

---

### 🎯 **Метрики GRPO (Group Relative Policy Optimization)**

| Метрика | Назначение | Где используется | Целевое значение |
|---------|-----------|------------------|------------------|
| **Group Mean Reward** | Средняя награда по группе вариантов | Оптимизация сравнительного обучения | Растет со временем |
| **Group Std Reward** | Стандартное отклонение награды в группе | Измеряет разнообразие качества | Оптимальное > 0 |
| **Advantage** | Относительное преимущество варианта vs среднее | Выбор лучшего варианта из группы | > 0 = выше среднего |
| **Rank in Group** | Ранг варианта внутри батча (1 = лучший) | Сортировка для выбора | 1 |

---

### 📈 **Метрики по Типам Задач (Task Type Distribution)**

| Метрика | Назначение | Где используется | Примеры типов |
|---------|-----------|------------------|---|
| **Count** | Количество вариантов по типу | Балансировка датасета | crypto_text_web, forensics_image, xss_injection |
| **Percentage (%)** | Доля от всех вариантов | Распределение работы генератора | 25%, 35%, 40% |
| **Pass Rate по типу (%)** | Качество для каждого типа задачи | Определение проблемных типов | > 70% для каждого |
| **Avg Quality по типу /10** | Среднее качество по типу | Тюнинг промптов для каждого типа | > 7.5 для каждого |

---

### 🎪 **Метрики по Сложности (Quality by Difficulty)**

| Метрика | Назначение | Где используется | Диапазоны |
|---------|-----------|------------------|-----------|
| **Generated Count** | Кол-во сгенерированных задач на уровне сложности | Контроль баланса сложности | Easy, Medium, Hard |
| **Pass Rate по сложности (%)** | % прошедших на каждом уровне | Оценка адекватности сложности | Easy: > 80%, Medium: > 60%, Hard: > 40% |
| **Avg Quality по сложности /10** | Качество на каждом уровне | Исправление недостатков уровня | Должно быть > 7 для всех |

---

### 🌡️ **Метрики Температуры (Temperature Tuning Analysis)**

| Метрика | Назначение | Где используется | Оптимальный диапазон |
|---------|-----------|------------------|-----|
| **Temperature** | Параметр творчества LLM (0-2) | Контроль разнообразия vs стабильность | 0.7-1.2 |
| **Variants при T** | Кол-во попыток при данной температуре | Статистика экспериментов | > 50 для надежности |
| **Pass Rate при T (%)** | Успешность при данной температуре | Выбор оптимальной T | Макс 70-80% |
| **Avg Tokens при T** | Расход токенов при данной T | Оптимизация стоимости | Минимум для лучшего качества |

### 🧠 **Метрики RAG (Knowledge Base Integration)**

| Метрика | Назначение | Где используется | Целевое значение |
|---------|-----------|------------------|------------------|
| **RAG Enabled Count** | Батчей, использующих контекст из KB | Оценка эффективности RAG | > 50% |
| **RAG Percentage (%)** | % батчей с RAG контекстом | Показывает интеграцию KB | > 40% |
| **RAG Context IDs** | Какие KB статьи использовались | Отслеживание источников | Логирование для аудита |
| **RAG Context Summary** | Краткое описание добавленного контекста | Объяснение влияния KB | Для отладки |

### ❌ **Метрики Отказов (Top Rejection Reasons)**

| Метрика | Назначение | Где используется | Целевое значение |
|---------|-----------|------------------|------------------|
| **Failure Reason** | Причина, почему задача не прошла | Улучшение пайплайна | Минимум причин |
| **Count** | Кол-во отказов по этой причине | Приоритизация исправлений | < 5% от батча |
| **Percentage (%)** | Доля этой причины от всех отказов | Анализ частотности | Top 3 = 80% проблем |

### 📅 **Метрики Истории (Monthly Token Usage)**

| Метрика | Назначение | Где используется | Тренд |
|---------|-----------|------------------|-------|
| **Month** | Месяц анализа | Отслеживание trends | Последние 3 месяца |
| **Task Count** | Задач в месяц | Оценка нагрузки | Растет |
| **Avg Tokens/Task** | Средний расход на задачу в месяц | Оптимизация затрат | Должен снижаться |

### 🔧 **Метрики Вознаграждения (Reward Scoring)**

| Метрика | Назначение | Где используется | Структура |
|---------|-----------|------------------|-----------|
| **Reward Checks** | Детальные проверки вознаграждения | JSONB объект с проверками | {check_name: bool, ...} |
| **Reward Total** | Сумма всех вознаграждений | Взвешенная оценка | Float, обычно 0-100 |
| **Reward Binary** | Бинарная оценка (прошла ли все проверки) | Быстрый фильтр качества | 0 или 1 |
| **Passed All Binary** | Флаг прохождения всех бинарных критериев | Определение пригодности | true/false |

---

### 💾 **Метрики Артефактов**

| Метрика | Назначение | Где используется | Содержит |

| **Quality Details** | Детальная оценка качества от LLM | Объяснение качества | JSON с критериями |

### 📝 **Метрики Отслеживания**

## 📊 Иерархия метрик

```
AI Generation Batch (запрос)
├── Generation Efficiency Metrics
│   ├── Completion Rate
│   ├── Avg Generation Time
│   └── RAG Usage %
│
└── N Variants (варианты)
    ├── Quality Metrics
    │   ├── Pass Rate (Binary)
    │   ├── Quality Score
    │   └── Artifact Result
    │
    ├── Reward Metrics (GRPO)
    │   ├── Reward Total
    │   ├── Reward Binary
    │   ├── Advantage
    │   └── Rank in Group
    │
    └── Selection & Publishing
        ├── Is Selected
        ├── Published Task ID
        └── Failure Reason
```

## 🎯 Как использовать метрики

### Для аналитики качества:
```sql
SELECT task_type, difficulty, pass_rate, avg_quality
FROM ai_generation_variants
WHERE created_at > NOW() - INTERVAL '7 days'
ORDER BY pass_rate DESC;
```

### Для оптимизации стоимости:
```sql
SELECT temperature, COUNT(*), avg(tokens_input + tokens_output) as avg_tokens
FROM ai_generation_variants
GROUP BY temperature
ORDER BY avg_tokens;
```

### Для отладки отказов:
```sql
SELECT failure_reason, COUNT(*) as count
FROM ai_generation_variants
WHERE failure_reason IS NOT NULL
GROUP BY failure_reason
ORDER BY count DESC;
```

### Запуск полного отчета:
```bash
python scripts/generate_metrics_report.py
```

## 🔄 Feedback Loop цикл

```
Генерация → Вознаграждение → Аналитика → Улучшение промптов → Генерация
             (GRPO)           (Метрики)   (Temperature, RAG)
```

**Метрики питают GRPO оптимизацию для лучших результатов** ✨
